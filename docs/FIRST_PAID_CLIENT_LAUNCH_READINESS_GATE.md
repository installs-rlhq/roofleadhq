# First Paid Client Launch Readiness Gate

Date: 2026-06-05

## Purpose

Use this gate to make the final go/no-go decision before launching the first paid contractor in the RoofLeadHQ Founder-Led Launch Program.

This is the last checkpoint before the founder-led launch begins. It keeps the launch focused on review, monitoring, and helping the contractor book inspections / book appointments while production automations stay disabled unless separately approved.

Reference docs and tools:

- Launch packet: `docs/FIRST_PAID_PILOT_LAUNCH_PACKET.md`
- Setup checklist: `docs/FIRST_PAID_CONTRACTOR_SETUP_CHECKLIST.md`
- Kickoff email draft: `docs/FIRST_PAID_CONTRACTOR_KICKOFF_EMAIL_DRAFT.md`
- Aggregate read-only verifier: `node backend/scripts/verify-first-paid-pilot-readiness-readonly.js`
- Operator status page: `website/dashboard/pilot-status.html`

## Go Or No-Go Checklist

Mark every item complete before launch.

- Aggregate verifier passes
- Launch packet reviewed
- Contractor intake complete
- Setup checklist complete
- Kickoff email ready or sent
- Dashboard access ready
- Daily operations rhythm ready
- Manual Outreach expectations confirmed
- Vapi phone lead intake expectations confirmed
- Reporting expectations confirmed
- All live automation disabled
- Explicit approval required before any production automation

## Go Criteria

Proceed only if all of the following are true:

- The aggregate verifier reports `demo_ready_with_live_automation_disabled`.
- The operator status page is available and shows live automation disabled.
- The launch packet has been reviewed.
- The contractor intake worksheet is complete.
- The setup checklist is complete.
- The kickoff email is ready or already sent.
- Dashboard access is ready for the contractor.
- The daily operations rhythm is ready for founder-led monitoring.
- Manual Outreach expectations are confirmed.
- Vapi phone lead intake expectations are confirmed.
- Reporting expectations are confirmed.
- All live automation remains disabled.
- No production automation will run without explicit approval.

## No-Go Criteria

Do not launch if any of the following are true:

- The aggregate verifier does not pass.
- Any launch packet, intake, setup, or kickoff step is incomplete.
- Dashboard access is not ready.
- Daily operations rhythm is not ready.
- Manual Outreach expectations are unclear.
- Vapi phone lead intake expectations are unclear.
- Reporting expectations are unclear.
- Any live automation is enabled.
- Any production automation would run without explicit approval.
- Any hard outcome promise is being made about jobs, revenue, or fixed appointment volume.
